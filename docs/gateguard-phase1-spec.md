## 1. 风险评分矩阵接口定义

```typescript
interface GateRiskScore {
  // 总分 0-100，越高风险越大
  total_score: number;
  
  // 各维度分项得分
  dimensions: {
    operation_type: number;      // 操作类型风险 权重35%
    scope_impact: number;        // 影响范围 权重25%
    irreversibility: number;     // 不可逆程度 权重20%
    context_maturity: number;    // 上下文成熟度 权重15%
    historical_pattern: number;  // 历史模式匹配 权重5%
  };
  
  // 元数据
  metadata: {
    tool_name: string;
    tool_input: Record<string, any>;
    session_id: string;
    timestamp: number;
    scoring_version: string;
  };
  
  // 证据链
  evidence: Array<{
    dimension: string;
    factor: string;
    weight: number;
    score: number;
    description: string;
  }>;
}
```

✅ **接口验收标准**
- 幂等性：相同输入必须返回相同评分
- 可观测性：每个评分必须附带完整证据链
- 性能要求：单次评分耗时 < 10ms
- 可审计：所有评分可回溯可重放

## 2. 维度评分标准

| 维度 | 权重 | 0-20分 | 21-40分 | 41-60分 | 61-80分 | 81-100分 |
|---|---|---|---|---|---|---|
| 操作类型 | 35% | 只读操作 | 安全写入 | 文件编辑 | 创建文件 | 破坏性操作 |
| 影响范围 | 25% | 临时文件 | 单一文件 | 模块级 | 系统级 | 全局配置 |
| 可逆性 | 20% | 有未提交变更 | git已跟踪 | 新建文件 | 覆盖修改 | 不可逆删除 |
| 上下文 | 15% | 已完整调查 | 部分读取 | 已读部分 | 无上下文 | 首次操作 |
| 历史模式 | 5% | 已批准模式 | 常见模式 | 普通 | 罕见 | 异常 |
